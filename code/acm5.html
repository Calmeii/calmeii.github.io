swap用于交换两个变量

void swap(int &a,int &b)

  if (a != b) 

    {

      a ^= b;

      b ^= a;

      a ^= b;

    }

a^=b等价于a = a^b

其中^是位异或运算，即将a与b的对应位进行异或运算，同为0或者同为1时，对应位结果为0；否则为1。

比如，假设，a的值为二进制的1010，b的值为二进制的1100，那么a^b = 0110

以a=7，b=5为例，a=111，b=101

a^=b a=010

b^=a b=111

a^=b a=101

不难看出，就是巧妙的利用了01互补的方法实现了换位

枚举法

通过列举每一条数据进行验证，得出最后结论的方法。

思想非常简单，应用非常广泛

所谓枚举法,指的是从可能的解集合中一一枚举各元素,用题目给定的检验条

件判定哪些是无用的,哪些是有用的.能使命题成立,即为其解。一般思路：

对命题建立正确的数学模型；



根据命题确定的数学模型中各变量的变化范围（即可能解的范围）；



利用循环语句、条件判断语句逐步求解或证明；

枚举法的特点是算法简单，但有时运算量大。对于可能确定解的值域又一时

找不到其他更好的算法时可以采用枚举法。

又如

求满足表达式A+B=C的所有整数解，其中A，B，C为1~3之间的整数。

分析：本题非常简单，即枚举所有情况，符合表达式即可。算法如下：

for (int A=1;A<=3;A++)

  for (int B=1;B<=3;B++)

    for (int C=1;C<=3;C++)

      if (A+B==C) 

        printf("%d+%d=%d\n",A,B,C);

上例采用的就是枚举法。所谓枚举法，指的是从可能的解的集合中一一枚举

各元素，用题目给定的检验条件判定哪些是无用的，哪些是有用的。能使命

题成立的，即为解。

从枚举法的定义可以看出，枚举法本质上属于搜索。但与隐式图的搜索有所

区别，在采用枚举法求解的问题时，必须满足两个条件：

① 预先确定解的个数n；

② 对每个解变量A1，A2，……，An的取值，其变化范围需预先确定

A1∈{X11，……，X1p}

……

Ai∈{Xi1，……，Xiq}

……

An∈{Xn1，……，Xnk}

例中的解变量有3个：A，B，C。其中

A解变量值的可能取值范围A∈{1，2，3}

B解变量值的可能取值范围B∈{1，2，3}

C解变量值的可能取值范围C∈{1，2，3}

则问题的可能解有27个

（A，B，C）∈{（1，1，1），（1，1，2），（1，1，3），

（1，2，1），（1，2，2），（1，2，3），

………………………………

（3，3，1），（3，3，2），（3，3，3）}

在上述可能解集合中，满足题目给定的检验条件的解元素，即为问题的解。

如果我们无法预先确定解的个数或各解的值域，则不能用枚举，只能采用搜

索等算法求解。由于回溯法在搜索每个可能解的枚举次数一般不止一次，因

此，对于同样规模的问题，回溯算法要比枚举法时间复杂度稍高。

给你n根火柴棍，你可以拼出多少个形如“A+B=C”的等式？等式中的A、B、C是用火柴棍拼出的整数（若该数非零，则最高位不能是0）。用火柴棍拼数字0-9的拼法如图所示：

注意：

  1. 加号与等号各自需要两根火柴棍

  2. 如果A≠B，则A+B=C与B+A=C视为不同的等式（A、B、C>=0）

  3. n根火柴棍必须全部用上

【输入】输入n共一行，又一个整数n（n<=24）。

【输出】输出共一行，表示能拼成的不同等式的数目。

【输入输出样例1】

  14  

  2

【输入输出样例1解释】

2个等式为0+1=1和1+0=1。

【输入输出样例2】

18

9

【输入输出样例2解释】

9个等式为：

0+4=4

0+11=11

1+10=11

2+2=4

2+7=9

4+0=4

7+2=9

10+1=11

11+0=11

分析:给定n根火柴,找出拼成A+B=C有多少种方案。显然我们可以枚举A和B，C=A+B,如果A、B和C所使用的火柴总数n-4 (n<=24，“+”和“=”要用去4根火柴)，则是一种方案。

A枚举的范围是多少呢？通过观察可以得到每个数字对应的火柴数如下：

最多有20根火柴可以用，要拼三个数A、B、C，数字“1”需要的火柴数为2，A最多可取到“1111”，需要8根火柴，C也至少需要8根火柴,而且B还需要火柴。由以分析可知A、B均不可能超过“1111”。

我们可以把2222以内的数所需的火柴数预处理出来。

#include<stdio.h>

int n,f[2305],a[15]={6,2,5,5,4,5,6,3,7,6},ans;//a[i]表示数字i需火柴数 

int main()

{

 scanf("%d",&n);

 for (int i=0;i<=2300;i++)//预处理出数i所需要火柴数f[i] 

   {

   int sum=0,m=i;

   do 

    {

     sum+=a[m%10];

     m=m/10;

    }

    while (m!=0);

   f[i]=sum;

   }

for (int i=0;i<=1111;i++)//枚举A和B 

 for (int j=0;j<=1111;j++)

  if (f[i]+f[j]+f[i+j]==n-4) ans++;//能拼成A+B=C 

   printf("%d\n",ans);

return 0;

}

hvalue hpow(hvalue r, int n)

  return v[n] = hpow(r, n/2) * hpow(r, (n+1)/2);

}
