Kmp算法

字符串匹配是计算机科学中最古老、研究最广泛的问题之一。

要求：在文本串 text[]中,找出其是否包含另一个字符串 pattern[]？

Input:
I am happy to join with you today in what will go down in history as the greatest demonstration for freedom in the history of our nation.
or fre

Output:
Yes,I can find it!

Input:
ababcabcaa
abcba

Output:
No,I can't find it!

经典的两种字符串匹配算法：

1.暴力匹配[Brute Force algorithm]

暴力算法是字符串匹配算法中最容易想到的算法

暴力算法就是文本串text[]每次移动向右移动一个字符，与模式串pattern[]比较，如果每个字符都相等，则表示在文本串中找到与模式串匹配的子集。

时间复杂度为O(lt*lp)　　　　[lt = strlen(text) , lp = strlen(pattern)]

2.Kmp算法

KMP算法是一种改进的字符串匹配算法，由D.E.Knuth，J.H.Morris和V.R.Pratt同时发现，因此人们称它为克努特——莫里斯——普拉特操作（简称KMP算法）。 KMP算法的关键是利用匹配失败后的信息，尽量减少模式串与主串的匹配次数以达到快速匹配的目的。具体实现就是实现一个next()函数，函数本身包含了模式串的局部匹配信息。

时间复杂度为O(lt + lp)

暴力匹配，成功匹配了一大串字符串，然而有一个字符不相同，就会退回从头比较。既然已经匹配了一部分子串，那么只需“指针”j 回溯到模式串已匹配过的子串中的某一个字符。
存在最长(此处长度为2)的前缀字符串“ab”与最长后缀字符串“ab”匹配，所以当T[7]与P[7]不相等时，直接后挪P[]对齐后缀“ab”,比较T[7]与P[2]

由上图 当 T[i]!=P[j] 时，
　　　　由于，T[i-j,....,i-1] == P[0,...,j-1];
　　　　若，P[0,...,k-1] == P[j-k,...,j-1];
则,
　　　　T[i-k,..,i-1] == P[0,...,k-1]

所以，当T[i]与P[j]不相等时，可以直接比较T[i]与P[k]，看其是否匹配；指针i 不用回溯，因此较暴力匹配而言，大大加快了比较速率

平移P[]得到next[]数组值，

"P[0,1]==P[5,6]",所以 在T[7]与P[7]失配时，下一次比较，i=7不必回溯，j=7回溯到j=2,即比较 T[7]与P[2],（P[7]与文本串失配时，用P[next[7]]与其匹配）

kmp的匹配过程：

T[i-1]==P[j-1]
T[i] != P[j]
T[i] ?= P[ k = next[j] ]

此处，举例子。。

模式串P的next[]值为{-1,0,0,1,2,0,1,2}。
next[j]的意义，P串前面i个字符所组成的串p，最长相等的 前缀与后缀 长度

eg: ababcaba, 当j=4时，最长 相等前缀与后缀 为"ab",即"p[0]p[1]"=="p[2]p[3]",长度为2.所以，next[4]==2

匹配过程如下：

当 T[7]!=P[7]时，比较T[7]?=P[ next[7] ],即 T[7]?=P[2]；
当 T[7]!=P[2]时，比较T[7]?=P[ next[2] ],即 T[7]?=P[0]；
当 T[7]!=P[0]时，比较T[7]?=P[ next[0] ],即 T[7]?=P[-1]；next[j]==-1时，不必比较，转为比较 T[7+1]?=P[-1+1]
当 T[8]==P[0]时，比较T[8+1]?=P[0+1]
当 T[9]==P[1]时，比较T[9+1]?=P[1+1]

kmp匹配过程的代码

int kmp()
{
    int i = 0, j = 0, count = 0;
    while (i != lt && j != lp)
    {
        if (T[i] == P[j] || j == -1)
            ++i, ++j;
        else
            j = next[j];
        if (j == lp)
        {
            count++;
            j = next[j];
        }
    }
    return count;
}

然后是解决怎样计算next函数值的问题，一个递推的过程

1）
已知： next[0]=-1;
假设： next[j]=k,
　　　　　　　　　　　　　如果 P[j]==P[k],
结论：　　　　　　　　　next[j+1]==k+1， ......A

(因为，由next[j]=k得，模式串P[]，在j之前，存在一个长度为k的后缀子串 与 k长度的前缀子串相同)

{证明：
　　　　next[j]=k;
　　　　所以，P[0,..,k-1] == P[j-k,..,j-1]
　　　　如果 P[k] == P[j]
　　　　那么 P[0,..,k] == P[j-k,..,j]
　　　　所以，next[j+1]==k+1
}

举个例子如下

由 next[6]==1, 知 前6个字符 最长前后缀子串长度为1， 因此得到 P[0]==P[5]。如果，P[6]==P[1],容易想到，next[7]=next[6]+1 = 2
依次类推，P[3]==P[8],next[9]=next[8]+1=4

2）
已知：　　　next[0]=-1;
假设:　　　　next[j]=k,
　　　　　　　　　　　如果 P[j]!=P[k],
那么

因为P[9]!=P[4],所以 next[10] != next[9]+1
k=next[k]， ......B

如果 P[j]==P[k] ,继续 1)A
如果 P[j]!=P[k] ,继续 2)B

(上图比较P[9]与P[2],相等,所以next[9+1]==2+1)

求取next[]数组的代码

void getnext()
{
    int i = 0, j = -1;
    next[0] = -1;
    while (i != lp)
    {
        if (j == -1 || P[i] == P[j])
            next[++i] = ++j;
        else
            j = next[j];
    }
}

到此，kmp算法基本结束

特殊情况：

next[]={-1,0,1,2,3}

T[i]!=P[j], T[i]?=P[k = next[j]]
若 P[j]==P[k]
则 T[i]!=P[k]

因此，　　　P[j]!=P[k] --->　　newnext[j]=k;
　　　　　　P[j]==P[k] --->　　newnext[j]=newnext[ k = newnext[j] ]

所以修改值newnext[]={-1,-1,-1,-1,3}

求取newnext[]数组的代码

void getnewnext()
{
    int i = 0, j = -1;
    newnext[0] = -1;
    while (i != lp)
    {
        if (j == -1 || P[i] == P[j]){
            ++i,++j;
            if(P[i]!=P[j])
                newnext[i]=j;
            else
                newnext[i]=newnext[j];
        else
            j = newnext[j];
    }
}
